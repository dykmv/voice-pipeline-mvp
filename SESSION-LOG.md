### 2. Working MVP Application (in product-mvp/)

**Tech stack:**
- Next.js 16 + React 19 + TypeScript
- Tailwind v4 + shadcn/ui (base-nova, dark theme)
- Prisma + PostgreSQL (Neon cloud)
- OpenAI API (gpt-4o-mini) for voice command parsing
- Web Speech API for speech-to-text
- NextAuth v4 with email + password auth
- Deployed on Vercel + Neon

**Screens:**
- `/` — Landing page (redirects to landing.html)
- `/login` — Sign in with email + password
- `/register` — Create account
- `/app` — Pipeline Board (Kanban with 6 columns, drag & drop)
- `/today` — Today View (follow-ups, viewings, overdue)
- `/leads/[id]` — Lead Detail (fields, status, timeline, edit)

**Key features:**
- **Conversational voice assistant** — bottom sheet dialog with multi-turn conversation
  - LLM parses free-form voice into structured actions
  - Assistant asks clarifying questions (property type, location, budget, follow-up date)
  - Smart quick-reply buttons generated by LLM
  - Confirmation flow: assistant summarizes → user confirms → action applies
  - Works on all screens (Pipeline, Today, Lead Detail with context)
- **Sticky input bar** — centered at bottom of every screen (type or tap mic)
- **Drag & drop** — lead cards between pipeline columns with comment dialog
- **Cyprus knowledge** — assistant knows cities, areas, streets; corrects transcription errors
- **Phone handling** — accepts any format, silently adds +357 for Cyprus
- **Auth** — email + password, bcrypt hashing, JWT sessions, route protection
- **Avatar menu** — initials circle + sign out dropdown

**Lead fields:**
name, phone, email, source, budget, propertyType, location, status, followUpDate,
transactionType (Sale/Rent), leadRole (Buyer/Seller/Tenant), nextAction,
viewingDate, refNumber

### 4. Key Decisions Made

1. **SQLite → PostgreSQL** — for Vercel serverless compatibility
2. **OpenAI over Claude API** — user had OpenAI key; blocked from RU server, works on Vercel
3. **Conversational assistant over simple confirm bar** — UX research showed agents want dialog, not command processing
4. **Bottom sheet on mobile, centered modal on desktop** — for the assistant dialog
5. **Smart LLM-generated quick replies** — instead of hardcoded Yes/No buttons
6. **Email+password over Google OAuth** — simpler, no Google Console setup needed
7. **No lead deletion** — leads only move between statuses (PRD decision)
8. **Placy design system (dark theme)** — from placy-website-design/ tokens
9. **No Placy branding** — product is "VoicePipeline", standalone experiment

### 5. UX Issues Identified & Fixed

- Auto-zoom on mobile inputs → viewport max-scale + 16px font-size
- Confirm bar too small → replaced with full conversational dialog
- "Yes" not working (stale closure bug) → useRef for messages
- Search showing no data → server-side DB query + second LLM call
- Lead detail missing assistant → AssistantDialog with lead context
- Speech timeout too short → 5s silence, 30s hard limit
- Phone over-validation → simplified, accept anything
